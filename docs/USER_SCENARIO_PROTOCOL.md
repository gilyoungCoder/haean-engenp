# Haean User Scenario Protocol (사용자 시나리오 프로토콜)

> Version: 1.0 | Date: 2026-03-31
> 이 문서는 Haean 웹 서비스의 사용자 관점 UX 흐름, 각 화면의 목적, 데이터 제한, 기본 설정값을 상세히 기술합니다.

---

## 1. 서비스 개요

Haean은 영어 내신 단과 강사를 위한 AI 변형문제 생성 서비스입니다.

**핵심 가치**: 시험 범위 지문(JPG/PNG/PDF) 1장 업로드 → AI 분석 → 변형 문제지 자동 생성 → .docx 다운로드

**3가지 생성 모드**:
| 모드 | 목적 | 사용 시나리오 |
|------|------|-------------|
| 변형문제 | 기존 지문으로 새로운 문제 유형 생성 | 내신 대비 보충 프린트 제작 |
| 워크북 | 빈칸, 어순배열 등 학습용 워크시트 생성 | 수업 중 활동지, 숙제용 |
| 동형모의 | 학교 기출 패턴을 분석하여 동일 스타일 모의고사 생성 | 내신 직전 모의고사 제작 |

---

## 2. 전체 사용 흐름 (End-to-End Flow)

```
[로그인] → [대시보드 진입] → [파일 업로드] → [생성 옵션 설정] → [문제 생성]
    → [결과 확인] → [AI 채팅으로 수정] → [DOCX 내보내기] → [프로젝트 저장]
```

### 2.1 인증 흐름

```
사용자 → /login 접속
  ├── Google 계정으로 로그인 (OAuth 2.0)
  └── 이메일/비밀번호 로그인 (선택)
      ↓
  세션 생성 → /dashboard 리다이렉트
```

- 카카오 로그인: 사업자 등록 완료 후 활성화 예정
- 세션 유지: NextAuth JWT, 브라우저 닫아도 30일간 유지

## 4. 메인 콘텐츠 3개 탭 상세

### 4.1 지문 원문 탭

**목적**: 업로드한 원본 파일을 그대로 보여줌
**표시 방식**:
- 이미지(JPG/PNG): `<img>` 태그로 base64 렌더링
- PDF: `<iframe>` 태그로 base64 data URI 렌더링
**데이터**: 가장 최근 업로드한 파일 1개만 표시
**제한**: 한 번에 1개 파일만 업로드 가능 (새 파일 업로드 시 이전 파일 교체)

### 4.2 타겟 지문 탭

**목적**: AI가 구조화 분석한 결과를 보여줌 (StructuredPassage)
**표시 항목**:
| 항목 | 설명 | 예시 |
|------|------|------|
| 제목 | AI가 추출한 지문 제목 | "The Impact of Social Media on Adolescents" |
| 전문 (Full Text) | 원문 텍스트 전체 | 영어 원문 |
| 단락별 문장 분석 | 각 단락의 문장 인덱스 + 텍스트 | [1] Social media has... [2] Studies show... |
| 핵심 어휘 | 단어, 품사, 정의, 동의어/반의어 | adolescent (n.) 청소년, synonyms: teenager |
| 문법 포인트 | 주요 문법 구조 | 관계대명사, 분사구문, 가정법 |
| 난이도 | 1-5 스케일 | Lv.3 |
| 단어 수 | 자동 카운트 | 287 words |
| 주제 | 키워드 태그 | ["social media", "psychology", "education"] |

**데이터**: 현재 프로젝트에서 가장 최근 구조화된 지문 1개
**갱신 시점**: `/api/structurize` 성공 후 자동 갱신

### 4.3 생성된 문제 탭

**목적**: AI가 생성한 문제 카드들을 나열
**표시 항목 (문제 카드)**:
| 항목 | 설명 |
|------|------|
| 문제 번호 | 순차 번호 (1, 2, 3...) |
| 유형 뱃지 | 색상 코딩된 유형 태그 (어법, 어휘, 빈칸추론 등) |
| 난이도 | Lv.1 ~ Lv.5 |
| 발문 (Instruction) | 문제 지시문 |
| 지문 (Passage with markers) | 밑줄/괄호 등 표시된 지문 |
| 선지 (Choices) | ①~⑤ 객관식 (해당 시) |
| 정답 (Answer) | 정답 |
| 해설 (Explanation) | 한국어 해설 (접기/펴기 토글) |
| 삭제 버튼 | 개별 문제 삭제 |

**누적 동작**: 새 파일 업로드 후 생성 시, 기존 문제에 **이어서** 추가됨 (덮어쓰기 아님)
**번호 매기기**: 기존 마지막 번호 다음부터 순차 부여

## 5. 핵심 시나리오별 상세 흐름

### 시나리오 A: 기본 변형문제 생성 (가장 일반적인 사용)

```
1. 사용자가 /dashboard 접속
2. 왼쪽 사이드바에서 [파일 업로드] 영역에 모의고사 이미지(JPG) 드래그 앤 드롭
   - 파일명이 업로드 영역 아래에 표시됨
   - 지원 형식: PDF, JPG, PNG (최대 10MB)
3. [생성 옵션]에서 설정:
   - 모드: "변형문제" 선택 (기본값)
   - 유형: 원하는 유형 체크 (예: 어법 선택 + 어휘 선택)
   - 난이도: 슬라이더로 Lv.1~Lv.5 조절 (기본값: Lv.3)
   - 문제 수: 슬라이더로 1~10 조절 (기본값: 유형당 3문항)
4. [문제 생성하기] 버튼 클릭
5. 프로그레스 표시:
   - "지문 분석 중... (1/2)" — structurize API 호출 (15-25초)
   - "문제 생성 중... (2/2)" — generate API 호출 (20-35초)
6. 완료 시:
   - [타겟 지문] 탭에 구조화된 지문 표시
   - [생성된 문제] 탭에 문제 카드들 표시
   - 토스트: "문제 생성이 완료되었습니다!"
7. 사용자가 문제 확인 후:
   - 마음에 안 드는 문제는 [삭제] 클릭으로 제거
   - AI 채팅에서 수정 요청 가능 (시나리오 D 참조)
8. [DOCX 내보내기] 버튼으로 Word 파일 다운로드
```

**소요 시간**: 약 35-60초 (파일 업로드 ~ 문제 생성 완료)

### 시나리오 B: 여러 지문으로 문제 누적 생성

```
1. 첫 번째 지문 업로드 → 어법(2문항) + 어휘(2문항) 생성
   → 생성된 문제: #1~#4 (4문항)

2. 두 번째 지문 업로드 (기존 파일 교체됨)
   → [지문 원문] 탭: 새 지문으로 교체
   → [생성된 문제] 탭: 기존 #1~#4 유지됨

3. [문제 생성하기] 클릭 → 빈칸추론(3문항) 생성
   → 생성된 문제: #1~#4 (기존) + #5~#7 (신규) = 총 7문항
   → [타겟 지문] 탭: 두 번째 지문의 구조화 결과로 교체

4. 세 번째 지문 업로드 → 문장삽입(2문항) 생성
   → 생성된 문제: #1~#7 (기존) + #8~#9 (신규) = 총 9문항

5. [DOCX 내보내기] → 전체 9문항이 하나의 Word 파일로 출력
```

**핵심 규칙**:
- 파일은 항상 1개만 (새 업로드 시 이전 교체)
- 문제는 누적됨 (이전 문제 + 새 문제)
- 타겟 지문은 가장 최근 구조화 결과만 표시
- 개별 문제 삭제 가능 (번호 자동 재정렬)

### 시나리오 C: 동형 모의고사 생성

```
1. [학교 DNA 분석] 드롭다운 열기
2. 학교 기출 시험지(PDF/이미지) 업로드
   - 여러 장 가능 (시험지 앞면/뒷면)
   - DNA 분석 API 호출 → SchoolDnaProfile 생성
   - 분석 결과: 출제 유형 분포, 문법 포인트 빈도, 난이도 패턴
3. 시험 범위 지문 업로드 (일반 업로드와 동일)
4. 생성 모드: "동형모의" 선택
5. [문제 생성하기] 클릭
   → DNA 프로필이 generate API에 주입됨
   → 학교 기출과 동일한 유형 비율/난이도로 문제 생성
6. 결과: 학교 시험과 유사한 형식의 모의고사 문제
```

**DNA 분석 제한**:
- 무료 플랜: DNA 분석 불가 (Pro 이상)
- 최대 이미지 수: 10장 (시험지 앞뒤면 5장)
- 분석 소요 시간: 20-40초

### 시나리오 D: AI 채팅으로 문제 수정

```
1. 문제가 생성된 상태에서 오른쪽 AI 채팅 사이드바 사용
2. 빠른 명령어 버튼 또는 직접 입력:
   - "난이도 올려줘" → AI가 수정 방안 제안
   - "문제 추가해줘" → AI가 추가 문제 유형 제안
   - "해설 자세히" → AI가 해설 보강
   - "다른 유형으로" → AI가 유형 변경 제안
3. AI 응답 흐름:
   ① AI가 수정 방법을 제안 (확인 요청)
   ② 사용자가 "네" / "좋아요" / "해줘" 등으로 확인
   ③ AI가 실제 수정 수행 → <questions_update> JSON 블록 반환
   ④ 대시보드에 수정된 문제가 자동 반영
4. 수정 규칙:
   - 기존 문제만 수정됨 (새 문제 추가 안 됨)
   - question_number 기준으로 매칭
   - 수정되지 않은 필드는 기존 값 유지 (deep merge)
```

**채팅 제한**:
- 무료 플랜: 월 50회 메시지
- 대화 기록: 프로젝트별 MongoDB에 저장
- 프로젝트 복원 시 이전 대화 기록 자동 로드

### 시나리오 E: 프로젝트 복원 (재방문)

```
1. 사용자가 다음 날 다시 로그인
2. 왼쪽 프로젝트 히스토리에서 이전 프로젝트 클릭
   - 프로젝트명: "2026-03-31 18:09" (생성 날짜시간)
3. 자동 복원되는 항목:
   ✅ 구조화된 지문 (StructuredPassage) — MongoDB에서
   ✅ 생성된 문제 (GeneratedQuestion[]) — MongoDB에서
   ✅ AI 채팅 기록 — MongoDB에서
   ✅ 원본 파일 (base64) — Cloudflare R2에서 다운로드
4. 복원 후 사용자는:
   - 기존 문제에 이어서 새 문제 생성 가능
   - AI 채팅으로 기존 문제 수정 가능
   - DOCX로 내보내기 가능
```

### 시나리오 F: 워크북 생성

```
1. 교과서 지문 이미지 업로드
2. 생성 모드: "워크북" 선택
3. 워크북 전용 유형 체크:
   - 한영 번역 (korean_to_english)
   - 번역 서술 (translation_writing)
   - 어순 배열 (word_order)
   - 어법 오류 (grammar_error_correction)
   - 전체 순서 (sentence_ordering_full)
   - 내용 일치 T/F (content_match_tf)
4. [문제 생성하기] 클릭 → 워크북 형식 문제 생성
5. DOCX 내보내기 → 학생 배포용 워크시트
```

## 6. 데이터 제한 및 기본 설정값

### 6.1 파일 업로드 제한

| 항목 | 제한 |
|------|------|
| 파일 형식 | PDF, JPG, PNG |
| 최대 파일 크기 | 10MB |
| 동시 업로드 | 1개 (새 파일 업로드 시 기존 교체) |
| 프로젝트당 파일 | 여러 번 교체 가능하나 최신 1개만 유지 |
| 스캔 PDF | 텍스트 50자 미만 시 거부 (이미지로 재업로드 안내) |

### 6.2 문제 생성 기본값

| 항목 | 기본값 | 범위 | 설명 |
|------|--------|------|------|
| 생성 모드 | 변형문제 | 변형문제 / 워크북 / 동형모의 | 탭으로 전환 |
| 난이도 | Lv.3 | Lv.1 ~ Lv.5 | 슬라이더 조절 |
| 유형당 문제 수 | 3문항 | 1 ~ 10 | 슬라이더 조절 |
| 선택 유형 수 | 제한 없음 | 1 ~ 10개 유형 | 체크박스 다중 선택 |
| 총 문제 수 상한 | 10문항/요청 | 자동 조절 | 60초 타임아웃 방지 |
| 해설 언어 | 한국어 | - | 고정 |

### 6.3 난이도 스케일 상세

| 레벨 | 설명 | 대상 학생 | 문제 특성 |
|------|------|----------|----------|
| Lv.1 | 매우 쉬움 | 중학교 수준, 기초 학습자 | 직접적 내용 확인, 단순 어휘, 명시적 답 |
| Lv.2 | 쉬움 | 고1 수준, 중하위권 | 기본 문법, 직관적 선지, 해석 중심 |
| Lv.3 | 보통 (기본값) | 고2 수준, 중위권 | 간접 추론, 중급 어휘, 문맥 파악 필요 |
| Lv.4 | 어려움 | 고3 수준, 상위권 | 복합 문법, 고급 어휘, 함정 선지 포함 |
| Lv.5 | 매우 어려움 | 수능 킬러, 최상위권 | 추상적 추론, 변별력 극대화, 고난도 문법 |

### 6.4 유형별 상세 (변형문제 모드)

| 유형 ID | 이름 | 난이도 범위 | 선지 | 발문 예시 |
|---------|------|-----------|------|----------|
| `grammar_choice` | 어법 선택 | Lv.2~5 | ①~⑤ | "다음 글의 밑줄 친 부분 중 어법상 틀린 것은?" |
| `vocabulary_choice` | 어휘 선택 | Lv.2~5 | ①~⑤ | "밑줄 친 단어와 의미가 가장 가까운 것은?" |
| `blank_inference` | 빈칸 추론 | Lv.3~5 | ①~⑤ | "빈칸에 들어갈 말로 가장 적절한 것은?" |
| `sentence_ordering` | 순서 배열 | Lv.3~5 | ①~⑤ | "주어진 글 다음에 이어질 글의 순서로 적절한 것은?" |

## 8. 프로젝트 생명주기

```
[생성] ──── [활성] ──── [저장됨] ──── [복원] ──── [삭제]
  │           │           │            │           │
  파일 업로드   문제 생성     자동 저장      히스토리 클릭  삭제 버튼
  Passage 생성  QuestionSet  MongoDB에      R2에서 base64  R2 파일 삭제
               ChatMessage   지속           다운로드       MongoDB 삭제
```

### 프로젝트 자동 저장 시점
- 파일 업로드 시 → Passage 레코드 생성
- 구조화 완료 시 → structuredData 저장
- 문제 생성 완료 시 → QuestionSet 저장
- AI 채팅 시 → ChatMessage 저장 (각 메시지마다)

### 프로젝트 복원 시 로드되는 데이터
| 데이터 | 출처 | 소요 시간 |
|--------|------|----------|
| 구조화 지문 | MongoDB (Passage.structuredData) | <1초 |
| 생성된 문제 | MongoDB (QuestionSet.questions) | <1초 |
| AI 채팅 기록 | MongoDB (ChatMessage) | <1초 |
| 원본 파일 | Cloudflare R2 (base64 다운로드) | 1-3초 |

### 프로젝트명 규칙
- 형식: `YYYY-MM-DD HH:mm` (예: "2026-03-31 19:05")
- 파일 업로드 시점의 서버 시간 기준
- 히스토리에서 프로젝트명 + 문제 수로 식별

## 9. 기대 사용 패턴 (Teacher Persona)

### 패턴 1: 내신 대비 프린트 제작 (가장 일반적)

```
강사 김OO — 고2 내신 대비반 운영

1. 교과서 Lesson 5 본문 캡처 (JPG)
2. 변형문제 모드 → 어법(3) + 어휘(3) + 빈칸추론(2) 선택
3. 난이도 Lv.3 → 문제 생성 (총 8문항)
4. AI 채팅: "3번 문제 난이도 좀 낮춰줘" → 수정 반영
5. DOCX 내보내기 → 프린트로 인쇄 → 수업 시간 배포

소요 시간: 약 3분 (기존 수작업 대비 90% 절감)
```

```
강사 이OO — 고3 내신 직전반

1. 학교 기출 시험지 3회분 업로드 → DNA 분석

소요 시간: 약 5분 (기존 직접 출제 대비 95% 절감)
```

```
강사 박OO — 고1 기초반
```

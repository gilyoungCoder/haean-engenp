## 1. 이 작업이 왜 필요한가?

변형 문제를 만들려면 기출 지문, 선지, 정답, 원문 출처를 먼저 정리해야 합니다.  
이걸 수작업으로 하면 30문항 기준 **1~2시간**이 걸리지만,  
Claude AI에게 파일을 통째로 넘기면 **10분 안에 구조화된 JSON**으로 돌아옵니다.

**이 작업으로 할 수 있는 것:**
- 기출 PDF → 문제·지문·선지·정답 자동 추출
- 시험 범위 원문 파일과 자동 매칭 (패러프레이징 변형도 인식)
- 밑줄, 빈칸, `(A)`, `ⓐ` 등 특수 기호 보존
- 정답 및 정답 근거(로직) AI 자동 도출
- 출제 오류 의심 항목 자동 플래그

---

## 2. 전체 작업 흐름

```
STEP 1  파일 준비
        기출 PDF + 시험 범위 원문 파일 수집 및 이름 정리
           ↓
STEP 2  Claude에 업로드 + 프롬프트 입력
        (claude.ai 대화창에 파일 첨부 후 지시문 붙여넣기)
           ↓
STEP 3  Claude 자동 처리
        문제 추출 → 원문 매칭 → 정답 도출 → JSON 생성
           ↓
STEP 4  교사 검토
        정답·로직 확인 / 특수 기호 보존 여부 / 오류 의심 항목
           ↓
STEP 5  피드백 → 최종 JSON 다운로드
```

> **팁:** 30문항을 한 번에 처리하면 응답이 끊길 수 있습니다.  
> 15문항씩 나눠서 요청하면 더 안정적입니다.

## 3. 파일 준비 방법

### 3-1. 기출문제 파일

| 항목 | 내용 |
|------|------|
| 형식 | PDF 권장, 텍스트(.txt) 병행 업로드 시 인식률 향상 |
| 2단 PDF | 프롬프트의 자동 감지 지시문이 처리. 감지 결과가 틀렸다면 바로 정정 가능 |
| 스캔본 | 이미지 PDF는 특수 기호 누락 가능 → txt 파일 필수 병행 |

> ⚠️ **주의:** 스캔 PDF만 단독 업로드하면 `①②`, `ⓐⓑ`, 빈칸 등이 누락됩니다.  
> 텍스트 파일(.txt)을 반드시 함께 올리세요.

### 3-2. 시험 범위 원문 파일

- 범위별로 파일을 구분합니다. (예: `범위1`, `범위2`, `범위3`...)
- PDF와 텍스트(.txt)를 **쌍으로** 준비하는 것을 권장합니다.
- 한글 해석이 포함된 파일은 파일명에 `한글_해석은_무시`를 추가하고,  
  프롬프트에도 **"영어 원문만 참고하라"** 고 명시해야 합니다.

### 3-3. 파일명 권장 규칙

```
{학교명}_{파일역할}_{내용}.pdf / .txt

예) 신목고_영어기출.pdf
    신목고_범위2_모의고사2_한글_해석은_무시.txt
    신목고_범위3_교과서.pdf
```

파일명이 명확할수록 Claude가 각 파일의 역할을 혼동 없이 파악합니다.

## 4. 프롬프트 작성 팁

### 4-1. 기본 프롬프트 템플릿

아래를 복사해서 파일명·문항 수만 수정하여 사용하세요.

```
[파일 설명]
* '{기출파일명}.pdf' 파일은 기출문제야.
  이 PDF가 2단(좌/우) 구성인지 1단(단일 열) 구성인지 먼저 스스로 판단해.
  - 판단 기준: 각 페이지에서 문제 번호나 지문 텍스트가
    좌·우 두 열로 나뉘어 배치되어 있으면 2단, 한 열로 흐르면 1단이야.
  - 2단으로 판단한 경우: 왼쪽 단을 위에서 아래로 전부 읽은 뒤,
    오른쪽 단을 위에서 아래로 읽는 순서로 처리해.
  - 1단으로 판단한 경우: 위에서 아래로 순서대로 읽으면 돼.
  - 작업을 시작하기 전에 "이 PDF는 [2단 / 1단] 구성으로 판단했습니다."라고
    한 줄 먼저 출력해줘.
* '{범위파일명}'.txt 등 파일들은 시험 범위 원문 지문이야.
* 파일명에 '한글_해석은_무시'라고 적힌 txt 파일은 영어 원문만 참고해.

[수행할 작업]
1. 문제 추출: 총 N문제(객관식 1~M번, 서답형 1~K번)를 추출해.
   단, 하나의 지문에 여러 문제가 딸린 경우 문맥이 끊기지 않게 처리해.

2. 원문 매칭: 각 문제에 대해 시험 범위 파일에서 원문 지문을 찾아 매칭해.
   패러프레이징(변형)된 경우도 파악하고,
   원문을 찾을 수 없으면 "해당없음"으로 표기해.

3. 특수 기호 보존: ①②, ⓐⓑ, (A)(B), 빈칸 등을 반드시 그대로 보존해.

4. 선지 및 정답 도출: 객관식은 1~5번 선지를 모두 추출하고
   정답과 정답 로직을 작성해. 서답형도 정답을 도출해.

5. 출력 형식: 반드시 아래 JSON 형식으로만 전체 결과를 출력해.
   서답형의 경우 "선지" 항목에는 "없음 (주관식)"이라고 적어.

[JSON 형식]
[
  {
    "문제번호": "1",
    "문제_유형": "내용 불일치 파악 (등등)",
    "기출_지문_텍스트": "...(특수기호 보존된 기출 지문)...",
    "원문_지문_파일명": "신목고_범위2_모의고사2.pdf",
    "선지": {
      "1": "선지 내용 1",
      "2": "선지 내용 2",
      "3": "선지 내용 3",
      "4": "선지 내용 4",
      "5": "선지 내용 5"
    },
    "정답": 3,
    "정답_로직": "이러이러해서 3번이 정답임"
  }
]
```

### 4-2. 지시 항목별 핵심 이유

| 지시 항목 | 왜 필요한가? | 빠뜨리면? |
|-----------|-------------|-----------|
| **레이아웃 자동 감지** | 2단 PDF는 좌우가 교차되면 내용이 뒤섞임. 자동 감지 후 판단 결과를 먼저 출력하게 하면 오류를 즉시 확인 가능 | 2단인데 1단으로 읽으면 문제 번호가 뒤죽박죽 추출됨 |
| **한글 해석 무시** | 해석이 섞이면 영어 원문 정확도 하락 | 한글 해석을 지문으로 오인 가능 |
| **패러프레이징 인식** | 기출문제는 원문을 변형한 경우가 많음 | 원문 매칭 실패, 해당없음 오처리 |
| **특수 기호 보존** | 빈칸·`(A)`·`ⓐ` 등이 문제 유형을 결정함 | 변형 문제 제작 불가능 |
| **JSON 형식 명시** | 형식 미지정 시 Claude가 임의 형식 사용 | 후속 자동화 불가 |

### 4-3. 추가 팁

- **레이아웃 자동 감지 확인법:** Claude가 작업 시작 전 `"이 PDF는 2단 구성으로 판단했습니다."` 또는 `"1단 구성으로 판단했습니다."`를 출력합니다. 만약 판단이 틀렸다면 `"실제로는 2단이야. 다시 읽어줘."`라고 바로 정정하면 됩니다.
- **자동 감지가 어려운 경우:** 좌·우 열 폭이 비슷하지 않거나 이미지 PDF인 경우 판단이 불안정할 수 있습니다. 이때는 기존 방식처럼 프롬프트에 직접 명시하는 것이 안전합니다.
- **그래프·도표 문항**은 시각 자료라 텍스트 추출이 불가합니다.  
  해당 문항 정답은 교사가 직접 입력하세요.
- **서답형 조건이 복잡한 문항**은 프롬프트에 "보기 단어 사용 여부를 정답_로직에 체크리스트로 포함해줘"를 추가하면 검토가 쉬워집니다.
- 프롬프트를 `.txt`로 저장해두면 다음 시험 때 파일명만 바꿔 재사용할 수 있습니다.

## 5. 출력 JSON 구조 이해

### 5-1. 각 필드 설명

| 필드명 | 설명 | 예시 |
|--------|------|------|
| `문제번호` | 객관식: 숫자 / 서답형: "서답형N" | `"1"`, `"서답형3"` |
| `문제_유형` | 출제 유형 명칭 | `"내용 불일치 파악"` |
| `기출_지문_텍스트` | 특수 기호 보존된 기출 지문 원문 | `"...it avoids ①it..."` |
| `원문_지문_파일명` | 매칭된 시험 범위 파일명 + 문항 번호 | `"신목고_범위2.pdf (36번)"` |
| `선지` | 객관식: `{"1":…,"5":…}` / 서답형: `"없음 (주관식)"` | |
| `정답` | 정답 번호(객관식) 또는 정답 텍스트(서답형) | `3` 또는 `"however"` |
| `정답_로직` | AI 도출 정답 근거 **(검토 필수)** | `"지문에서 ~라고 명시됨…"` |

### 5-2. 특수 케이스 처리

| 케이스 | 처리 방법 |
|--------|-----------|
| **복수 정답** ("두 개 고르면") | `"정답"` 필드에 `"①④ (1번과 4번)"` 형식으로 기재 |
| **출제 오류 의심** | `"정답"` 필드에 오류 내용 함께 명시 |
| **원문 없음** | `"원문_지문_파일명"` 에 `"해당없음 (외부 지문)"` 표기 |
| **그래프·도표** | `"기출_지문_텍스트"` 에 `"(시각 자료 — 텍스트 추출 불가)"` 표기 |
| **서답형 영작** | `"정답"` 에 모범 답안, `"정답_로직"` 에 보기 단어 체크 포함 |

## 6. 검토 및 피드백 방법

### 6-1. 반드시 확인할 5가지

1. **정답 로직 정확성** — 지문과 직접 대조하여 확인. AI는 그럴듯한 설명을 쓰더라도 틀릴 수 있습니다.
2. **특수 기호 보존** — `①②`, `ⓐⓑ`, `(A)(B)`, 빈칸이 원본과 동일한지 확인.
3. **복수 정답 문항** — "두 개 고르면" 유형은 정답이 반드시 두 개인지 확인.
4. **원문 매칭 정확성** — 교과서 본문과 모의고사 지문이 혼재하는 경우 특히 주의.
5. **서답형 조건 충족** — 보기 단어 수, 문장 수, 어법 조건이 모두 충족되었는지 확인.

### 6-2. 오류 발견 시 피드백 방법

JSON 출력 후 오류가 발견되면 대화창에서 명확하게 피드백합니다.

**피드백 형식:**
```
[문제번호]번: [오류 내용] → [수정 방향]
```

**실제 피드백 예시 — 정답 로직 모순:**
```
16번: 정답 로직에서 ⓔ를 '일치'로 분류했는데
불일치 목록에 포함하는 모순이 있어.
실제 불일치는 ⓐ,ⓑ인데 해당 선지가 없으니,
선지 오류로 표기하고 ②(ⓐ,ⓓ)를 정답 처리해.
```

**실제 피드백 예시 — 할루시네이션 제거:**
```
서답형 10번: 마지막 조합 단어 부분은 삭제해.
시험지 원본의 번호 위치가 텍스트 파일에 없어서
조합 불가하다고만 써줘.
```

### 6-3. 최종 JSON 추출

모든 피드백을 마친 후:
```
이걸 반영해서 최종 json을 추출해줘.
```
Claude가 수정된 JSON 파일을 생성하여 다운로드 링크로 제공합니다.

## 7. 주요 오류 사례 및 대처법

| 오류 유형 | 발생 원인 | 대처법 |
|-----------|-----------|--------|
| **정답 로직 내적 모순** | AI가 분석과 결론을 따로 생성하여 불일치 | 로직 텍스트와 정답 번호 교차 확인 후 피드백 |
| **그래프 문항 정답 오류** | 시각 자료는 이미지, 텍스트 추출 불가 | 해당 문항 정답은 교사가 직접 입력 |
| **할루시네이션** | 추론 불가 정보를 임의로 생성 | "원본 확인 필요" 처리 요청. 근거 없는 내용은 삭제 지시 |
| **특수 기호 누락** | PDF 인식 한계, 특히 이미지 PDF | txt 파일 병행 업로드로 보완 |
| **원문 매칭 오류** | 패러프레이징이 심하거나 외부 지문 | "해당없음" 플래그 후 교사가 직접 확인 |
| **2단 PDF 순서 혼재** | 자동 감지 실패 또는 이미지 PDF | Claude의 판단 결과 출력을 확인 후 틀렸으면 즉시 정정. 이미지 PDF는 직접 명시 권장 |
| **서답형 조건 미충족** | 보기 단어 수, 문장 수 등 조건 누락 | 프롬프트에 "조건 체크리스트를 로직에 포함" 명시 |

>
> Claude AI는 높은 정확도로 작업을 수행하지만,  
> **정답 로직 오류·할루시네이션·시각 자료 오답** 등이 발생할 수 있습니다.  
> 특히 정답과 정답 로직은 교사가 최종 검토한 후 사용하세요.  
> AI 출력물을 무검토로 학생에게 제공하지 않도록 합니다.
